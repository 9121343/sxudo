🧠 Connect Your Local SXUDO Model

📍 Current Situation

You're running ollama run sxudo which is great for terminal chat, but your web app needs the Ollama API server to connect.

🔧 Step 1: Stop Interactive Mode

In your terminal where ollama run sxudo is running, press Ctrl+C to exit.

🚀 Step 2: Start Ollama API Server

Run this command to start the API server that your web app can connect to:

set OLLAMA_HOST=0.0.0.0:11434 ollama serve

🌐 Step 3: Find Your IP Address

In another terminal, find your computer's IP address:

ipconfig

Look for IPv4 Address (e.g., 192.168.1.100)

⚙️ Step 4: Configure in SXUDO App

  1. Go to your SXUDO app
  2. Click ⚙️ Settings
  3. Scroll to "Ollama Configuration"
  4. Enter your IP address (not 127.0.0.1)
  5. Port: 11434
  6. Click "Connect to Ollama"

✅ Step 5: Test Your SXUDO Model

Once connected, your app will automatically use your custom SXUDO model with the personality you've defined!

The status should change from "📝 Demo Mode" to "🤖 AI Mode Active"

🔄 Alternative: Test SXUDO Model Locally

You can also test your model directly:

# Test your model via API curl -X POST http://localhost:11434/api/chat -H "Content-Type: application/json" -d '{ "model": "sxudo", "messages": [{"role": "user", "content": "Hello SXUDO!"}], "stream": false }'
